# <a name="creating-mobile-apps-with-xamarinforms-book"></a>Criando aplicativos móveis com o livro Xamarin.Forms

[![Baixar](~/media/shared/download.png) amostra Baixar a amostra](https://github.com/xamarin/xamarin-forms-book-samples)

<img src="images/cover-sml.png" title="Criação de aplicativos móveis com Xamarin.Forms Book" align="left" />O livro *Criando Aplicativos Móveis com Xamarin.Forms* de Charles Petzold é um guia para aprender a escrever aplicativos Xamarin.Forms. O único pré-requisito é o conhecimento da linguagem de programação C#. O livro fornece uma extensa exploração na interface de usuário Xamarin.Forms e também abrange animação, MVVM, gatilhos, comportamentos, layouts personalizados, renderizadores personalizados e muito mais.

O livro foi publicado na primavera de 2016, e não foi atualizado desde então. Há muito no livro que permanece valioso, mas parte do [material está desatualizado](#ways-in-which-the-book-is-outdated), e alguns tópicos não são mais inteiramente corretos ou completos.

## <a name="ways-in-which-the-book-is-outdated"></a>Maneiras pelas quais o livro está desatualizado

Desde a publicação do *Creating Mobile Apps com Xamarin.Forms,* vários novos recursos foram adicionados ao Xamarin.Forms. Esses novos recursos são descritos em artigos individuais na documentação [Xamarin.Forms.](/xamarin/)

Outras mudanças fizeram com que parte do conteúdo do livro fosse desatualizado:

### <a name="net-standard-20-libraries-have-replaced-portable-class-libraries"></a>Bibliotecas .NET Standard 2.0 substituíram bibliotecas de classe portáteis

Um aplicativo Xamarin.Forms geralmente usa uma biblioteca para compartilhar código entre as diferentes plataformas. Originalmente, esta era uma Biblioteca de Classe Portátil (PCL). Há muitas referências aos PCLs ao longo do livro e os resumos dos capítulos.

A Biblioteca de Classes Portáteis foi substituída por uma biblioteca .NET Standard 2.0, conforme descrito no artigo [.NET Standard 2.0 Support in Xamarin.Forms](~/xamarin-forms/internals/net-standard.md). Todo o código de [amostra](https://github.com/xamarin/xamarin-forms-book-samples) do livro foi atualizado para usar bibliotecas .NET Standard 2.0.

A maioria das informações do livro sobre o papel da Biblioteca de Classe Portátil permanece a mesma para uma biblioteca .NET Standard 2.0. Uma diferença é que apenas um PCL tem um "perfil" numérico. Além disso, existem algumas vantagens para bibliotecas .NET Standard 2.0. Por exemplo, o Capítulo 20, [Async e Arquivo I/O](summaries/chapter20.md) descreve como usar as plataformas subjacentes para a execução de I/O de arquivo. Isso não é mais necessário. A biblioteca .NET Standard 2.0 suporta as classes de [System.IO](xref:System.IO) familiar para todas as plataformas Xamarin.Forms.

A biblioteca .NET Standard 2.0 também permite que [`HttpClient`](xref:System.Net.Http.HttpClient) os aplicativos Xamarin.Forms usem para acessar arquivos pela Internet em vez de [`WebRequest`](xref:System.Net.WebRequest) outras classes.

### <a name="the-role-of-xaml-has-been-elevated"></a>O papel de XAML foi elevado

*A criação de aplicativos móveis com Xamarin.Forms* começa descrevendo como escrever aplicativos Xamarin.Forms usando C#. O Idioma de marcação de aplicativos extensíveis (XAML) não é introduzido até [o Capítulo 7. XAML vs. Código](summaries/chapter07.md).

XAML agora tem um papel muito maior em Xamarin.Forms. Os modelos de solução Xamarin.Forms distribuídos com o Visual Studio criam arquivos de página baseados em XAML. Um desenvolvedor que usa Xamarin.Forms deve se familiarizar com o XAML o mais cedo possível. A seção [eXtensible Application Markup Language (XAML)](~/xamarin-forms/xaml/index.yml) da documentação Xamarin.Forms contém vários artigos sobre XAML para que você comece.

### <a name="supported-platforms"></a>Plataformas compatíveis

Xamarin.Forms não suporta mais windows 8.1 e Windows Phone 8.1.

O livro às vezes faz referências ao _Windows Runtime_. Este é um termo que engloba a API do Windows usada em várias versões do Windows e Windows Phone. Versões mais recentes do Xamarin.Forms se restringem a suportar a Universal Windows Platform, que é a API para Windows 10 e Windows 10 Mobile.

Uma biblioteca .NET Standard 2.0 não suporta nenhuma versão do Windows 10 Mobile. Portanto, um aplicativo Xamarin.Forms usando uma biblioteca .NET Standard não será executado em um dispositivo Windows 10 Mobile. Os aplicativos Xamarin.Forms continuam a ser executados na área de trabalho do Windows 10, versões 10.0.16299.0 e superior.

O Xamarin.Forms tem suporte à pré-visualização para as plataformas [Mac,](~/xamarin-forms/platform/other/mac.md) [WPF,](~/xamarin-forms/platform/other/wpf.md) [GTK#](~/xamarin-forms/platform/other/gtk.md)e [Tizen.](~/xamarin-forms/platform/other/tizen.md)
